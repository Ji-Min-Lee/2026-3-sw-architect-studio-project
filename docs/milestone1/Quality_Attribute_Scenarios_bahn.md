# Quality Attribute Scenarios — Signal Quality Feedback

**팀 / Team**: Blue Sky (3팀) | **마일스톤 / Milestone**: M1 | **마감 / Due**: 2026-06-09

---

## U-7: Signal Quality Feedback (Usability)

| 항목 | 내용 |
|------|------|
| **Source** | 사용자 (시계 측정 중) |
| **Stimulus** | 마이크 입력 신호가 정상 범위를 벗어남 — 신호 진폭이 임계값 이하이거나, 노이즈 레벨이 beat 피크 대비 과도하게 높은 상태 |
| **Artifact** | GUI 상태 표시 영역 |
| **Environment** | 실시간 운영 상태 (Live 모드), 신호 품질이 측정에 영향을 줄 수 있는 환경 |
| **Response** | GUI에 신호 품질 경고 메시지를 즉시 표시하고, 정상 복귀 시 자동으로 경고를 해제한다 |
| **Response Measure** | 비정상 신호 발생 후 **≤ 1초** 이내 경고 표시. 정상 복귀 후 **≤ 2초** 이내 경고 해제 |

**경고 조건 및 메시지:**

| 상황 | 감지 조건 | 표시 메시지 |
|------|---------|-----------|
| 신호 약함 | `burst_max < min_peak_threshold` 가 연속 N회 발생 | `⚠ Signal too weak` |
| 노이즈 과다 | `noise_floor / ref_peak > 0.3` (SNR 저하) | `⚠ Noisy signal` |

**시나리오:**

> 사용자가 시계를 마이크에 올려놓았으나 접촉이 불완전하거나 주변 소음이 심한 경우, 시스템은 측정값이 신뢰할 수 없는 상태임을 사용자에게 명시적으로 알려야 한다.
>
> 현재 코드는 신호가 약하면 beat 이벤트가 발행되지 않아 화면 값이 그냥 멈추고, 노이즈가 많으면 값이 불안정해지지만 **사용자는 이유를 알 수 없다.** 이 QA가 만족되면 사용자는 측정값을 신뢰할 수 없는 상황을 즉시 인지하고 시계 위치를 조정하거나 환경을 개선하는 조치를 취할 수 있다.

---

## Priority

| 항목 | 수준 | 근거 |
|------|------|------|
| **Importance** | High | 신호 품질 문제는 측정값 신뢰도에 직접 영향을 미침. 사용자가 경고 없이 잘못된 값을 정상으로 오인하면 시계 조정 판단 자체가 틀려지므로 중요도가 높음 |
| **Difficulty** | High | 최적 임계값이 시계 기종·마이크 감도·환경 소음에 따라 달라지며, 현재 God Object 구조에서 신호 품질 로직을 추가하려면 대규모 코드 수정이 필요하므로 구현 난이도가 높음 |

---

## Risk

| 리스크 | 설명 | 영향 |
|--------|------|------|
| 오감지 발생 | 신호가 약하거나 노이즈가 많을 때 beat 이벤트를 잘못 감지(false positive) | 경고 없이 신뢰할 수 없는 Rate/Amplitude 값이 화면에 표시됨 |
| 경고 미발생 | 시스템이 신호 약함을 감지하지 못한 채 측정 계속 진행 | 사용자가 잘못된 값을 정상 측정값으로 오인할 수 있음 |
| 임계값 설정 어려움 | `noise_floor / ref_peak` 비율 최적값이 시계 기종·마이크 감도·환경 소음에 따라 달라짐 | 고정 임계값으로는 오경보(false alarm) 또는 미경보(missed alarm) 발생 가능 |
| 구현 복잡도 | 신호 품질 판단 로직을 현재 God Object(`MainWindow`)에 추가 시 기존 코드 전체 수정 필요 | 구현 난이도 높고 side effect 위험 존재 |

## 실험 계획 / Experiment Plan

### 실험 1: 신호 강도 저하 시 경고 발생 시점 확인

| 항목 | 내용 |
|------|------|
| **목적** | 시계를 마이크에서 점진적으로 멀리할 때 어느 시점부터 측정값이 신뢰할 수 없게 되는지, 그리고 그 시점에 경고가 표시되는지 확인 |
| **방법** | 시계를 마이크 센서에서 1cm 단위로 거리를 늘리면서 Rate 값의 안정성과 beat 감지 성공률을 기록. 동시에 `Signal too weak` 경고 발생 여부 확인 |
| **완료 기준** | 경고가 발생하는 거리와 Rate 오차가 급격히 증가하는 거리가 일치하거나 경고가 선행함 |

---

### 실험 2: 주변 소음 주입 시 경고 반응 확인

| 항목 | 내용 |
|------|------|
| **목적** | 주변 소음이 증가할 때 `Noisy signal` 경고가 적절한 시점에 발생하는지 확인 |
| **방법** | 정상 측정 중 주변 소음(대화 소리, 책상 진동 등)을 단계적으로 추가하면서 경고 발생 시점과 Rate 불안정 시점을 비교 |
| **완료 기준** | 소음으로 인해 Rate 값이 불안정해지는 시점에 경고가 표시됨. 소음이 제거되면 경고가 2초 이내 해제됨 |

---

### 실험 3: 경고 해제 반응 시간 확인

| 항목 | 내용 |
|------|------|
| **목적** | 신호가 정상으로 복귀했을 때 경고가 지정된 시간(≤ 2초) 이내에 해제되는지 확인 |
| **방법** | 의도적으로 신호를 약하게 만들어 경고를 발생시킨 후, 시계를 정상 위치에 다시 올려놓고 경고 해제까지의 시간을 측정 |
| **완료 기준** | 신호 정상 복귀 후 2초 이내 경고 해제 확인. 3회 반복 측정에서 모두 만족 |
