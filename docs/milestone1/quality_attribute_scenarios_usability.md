## U-7: Signal Quality Feedback (Usability)

| 항목 | 내용 |
|------|------|
| **Source** | 마이크 입력 신호 |
| **Stimulus** | 마이크 입력 신호가 정상 범위를 벗어남 — 노이즈가 있거나, 신호가 없는 상태 |
| **Artifact** | GUI 상태 표시 영역 |
| **Environment** | 실시간 운영 상태 (Live 모드), 신호 품질이 측정에 영향을 줄 수 있는 환경 |
| **Response** | GUI에 신호 품질 경고 메시지를 즉시 표시하고, 정상 복귀 시 자동으로 경고를 해제한다 |
| **Response Measure** | 비정상 신호 발생 후 **≤ N초** 이내 경고 표시. 정상 복귀 후 **≤ M초** 이내 경고 해제 *(잠정, 실험으로 확정)* |

**경고 조건 및 메시지:**

| 상황 | 감지 조건 | 표시 메시지 |
|------|---------|-----------|
| 신호 없음 | N초 동안 beat 이벤트가 감지되지 않음 | `⚠ No signal` |
| 노이즈 과다 | 주변 소음 레벨이 beat 신호 대비 과도하게 높음 | `⚠ Noisy signal` |

**시나리오:**

> 사용자가 시계를 마이크에 올려놓았으나 접촉이 불완전하거나 주변 소음이 심한 경우, 시스템은 측정값이 신뢰할 수 없는 상태임을 사용자에게 명시적으로 알려야 한다.
>
> 현재 코드는 신호가 없으면 beat 이벤트가 발행되지 않아 화면 값이 그냥 멈추고, 노이즈가 많으면 값이 불안정해지지만 **사용자는 이유를 알 수 없다.** 이 QA가 만족되면 사용자는 측정값을 신뢰할 수 없는 상황을 즉시 인지하고 시계 위치를 조정하거나 환경을 개선하는 조치를 취할 수 있다.

---

## Priority

| 항목 | 수준 | 근거 |
|------|------|------|
| **Importance** | High | 신호 품질 문제는 측정값 신뢰도에 직접 영향을 미침. 사용자가 경고 없이 잘못된 값을 정상으로 오인하면 시계 조정 판단 자체가 틀려지므로 중요도가 높음 |
| **Difficulty** | High | 최적 임계값이 시계 기종·마이크 감도·환경 소음에 따라 달라지며, 현재 God Object 구조에서 신호 품질 로직을 추가하려면 대규모 코드 수정이 필요하므로 구현 난이도가 높음 |

---

## Risk

| 리스크 | 설명 | 영향 |
|--------|------|------|
| 오감지 발생 | 신호가 없거나 노이즈가 많을 때 beat 이벤트를 잘못 감지(false positive) | 경고 없이 신뢰할 수 없는 Rate/Amplitude 값이 화면에 표시됨 |
| 경고 미발생 | 시스템이 신호 없음을 감지하지 못한 채 측정 계속 진행 | 사용자가 잘못된 값을 정상 측정값으로 오인할 수 있음 |
| 임계값 설정 어려움 | 경고 발생 기준이 시계 기종·마이크 감도·환경 소음에 따라 달라짐 | 고정 기준값으로는 오경보(false alarm) 또는 미경보(missed alarm) 발생 가능 |

## 실험 계획 / Experiment Plan

### 실험: N, M 값 확정

| 항목 | 내용 |
|------|------|
| **목적** | 기존 A/C 이벤트 감지 결과를 기준으로, 비정상 신호 발생 후 경고까지의 시간(N)과 정상 복귀 후 경고 해제까지의 시간(M)을 실측하여 확정 |
| **방법** | ① 정상 측정 중 시계를 제거하거나 USB를 뽑아 신호를 차단하고, 차단 시점부터 경고가 표시되기까지의 시간(N) 측정. ② 시계를 다시 올려놓은 후 A/C 이벤트가 안정적으로 재감지되는 시점부터 경고가 해제되기까지의 시간(M) 측정. 각 3회 반복 |
| **완료 기준** | N, M 수치 확정 및 Response Measure에 반영. 3회 반복 측정에서 모두 동일한 범위 내 결과 확인 |

## Tactic / Pattern

| 전술 / 패턴 | 연결 근거 |
|------------|---------|
| **Heartbeat** | 기존 A/C 이벤트를 heartbeat로 간주. N초 동안 이벤트가 수신되지 않으면 신호 소실로 판단하여 경고 발행. 별도 감지 로직 없이 기존 감지 결과를 재사용 |
