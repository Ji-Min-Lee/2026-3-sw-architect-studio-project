# TimeGrapher 그래프 해설

**팀**: Blue Sky (Team 3) | **플랫폼**: Raspberry Pi 5 + Qt/C++

> 기계식 시계 도메인 지식과 함께 각 그래프의 모든 요소를 설명합니다.

---

## 기계식 시계 기초 지식

TimeGrapher를 이해하려면 기계식 시계가 어떻게 작동하는지 알아야 합니다.

**밸런스 휠 (Balance Wheel)**
시계의 심장. 앞뒤로 왕복 진동하며 시간을 나눕니다. 28800 BPH 시계는 1초에 4번 진동 (앞뒤 각각 2번 = 8회 이벤트).

**이스케이프먼트 (Escapement)**
밸런스 휠의 진동에 맞춰 기어열을 한 칸씩 해제하는 기구. 이 해제 순간이 음향 신호로 감지됩니다.

**A 이벤트 (Tic)** — 밸런스 휠이 한쪽 방향으로 이스케이프먼트를 건드리는 순간.
**C 이벤트 (Tac)** — 반대 방향으로 돌아오며 이스케이프먼트를 건드리는 순간.

하나의 Beat = A이벤트 + C이벤트. 28800 BPH에서 한 Beat = 125ms.

**Rate (오차율)**
시계가 하루에 얼마나 빠르거나 느린지. 단위: 초/일 (s/d). 0이 이상적이며, 통상 ±10 s/d 이내면 합격.

**Amplitude (진폭)**
밸런스 휠의 흔들림 각도. 도(°) 단위. 건강한 시계는 270~310°. 낮으면 메인스프링 장력 부족 또는 마찰 과다.

**Beat Error**
A→다음A 구간과 C→다음C 구간이 얼마나 불균등한지. 단위: ms. 0이 이상적이며, 0.5ms 이하면 잘 조정된 것.

**포지션 (Position)**
시계를 어느 방향으로 놓느냐에 따라 중력의 영향이 달라져 Rate와 Amplitude가 변합니다. NIHS 95-10 / ISO 3158 표준 포지션:

| 코드 | 프랑스어 원어 | 의미 |
|------|-------------|------|
| CH | Cadran Haut | 다이얼 위 (시계 앞면이 위를 향함) |
| CB | Cadran Bas | 다이얼 아래 (시계 앞면이 아래를 향함) |
| 9H | 9 Heures | 크라운 아래 (9시 방향이 위를 향함) |
| 6H | 6 Heures | 크라운 왼쪽 (6시 방향이 위를 향함) |
| 3H | 3 Heures | 크라운 위 (3시 방향이 위를 향함) |
| 12H | 12 Heures | 크라운 오른쪽 (12시 방향이 위를 향함) |

## 공통 상태 바

모든 탭 상단에 항상 표시됩니다:

```
POS CH  |  RATE +1.9 s/d  |  AMP 204°  |  ERR 0.2 ms  |  BPH 28800
```

- **POS** — 현재 측정 포지션 (보라색)
- **RATE** — 현재 Rate 오차. 초록=허용범위, 빨강=범위 초과
- **AMP** — 현재 진폭(°). 초록=정상, 주황=경계, 빨강=문제
- **ERR** — 현재 Beat Error(ms). 초록=정상, 빨강=문제
- **BPH** — 분당 비트 수 (보라색). Auto BPH 모드에서 자동 감지
- **진단 배지** (우상단): 초록=Excellent, 주황=Good, 빨강=Needs Service

## 1. Rate / Scope

![Rate/Scope](graphs/01-rate-scope.png)

**한 줄 요약**: 매 Beat마다 오차를 점으로 찍고, 실시간 음향 신호를 아래에 보여주는 메인 측정 화면.

### 상단 패널 — Timing Offset 산점도

| 요소 | 색상 | 의미 |
|------|------|------|
| 점 | 빨강 | Tic(A이벤트) 타이밍 오프셋 — 해당 Beat의 Tic이 예상 시각에서 얼마나 벗어났는지 (ms) |
| 점 | 파랑 | Toc(C이벤트) 타이밍 오프셋 |
| 선 | 초록 | 20 Beat 이동평균 트렌드선 |

- **X축**: Beat 번호 (누적)
- **Y축**: 타이밍 오프셋 (ms). 0 기준선 위 = 빠름, 아래 = 느림
- **좌상단 박스**: `mean` (평균 오차), `σ` (표준편차, 일관성 지표), `n` (총 Beat 수)

**해석 방법**:
- 점들이 0 근처에 빽빽하게 모여 있고 트렌드선이 수평 → 잘 조정된 시계
- 트렌드선이 위로 기울어짐 → 시계가 점점 빨라지는 중 (메인스프링 풀림)
- 빨강/파랑 점의 간격이 넓음 → Tic과 Toc의 불균형 (Beat Error 큼)

### 하단 패널 — Scope 뷰 (실시간 음향 신호)

| 요소 | 색상 | 의미 |
|------|------|------|
| 선 | 파랑 | Rectified 신호 — 음향 신호의 절댓값 (실제 소리 에너지) |
| 선 | 빨강 | 검출 트리거 임계값 |
| 수직 점선 | 초록 | A 이벤트(Tic) 감지 시각 |
| 수직 점선 | 빨강 | C 이벤트(Tac) 감지 시각 |
| 화살표 + 레이블 | 검정 | A→C 간격(ms), 계산된 진폭(°), 전체 Beat 주기(ms) |

- **X축**: 실제 시각 (초)
- **Y축**: 신호 강도 (Amplitude)

**해석 방법**:
- 초록/빨강 점선 사이의 간격 = A→C 간격. 이 값에서 진폭과 Beat Error를 계산
- 파랑 파형의 스파이크 높이 = 음향 신호 강도 (마이크 거리와 시계 조건에 따라 변함)
- 빨강 트리거선 위에 파랑 파형이 올라와야 이벤트 감지 가능. 파형이 트리거 아래면 미감지

## 2. Sound Print

![Sound Print](graphs/02-sound-print.png)

**한 줄 요약**: 매 Beat의 음향 이벤트를 점으로 찍어 시계의 음향 지문(fingerprint)을 만드는 디스플레이.

### 범례

| 요소 | 색상 | 의미 |
|------|------|------|
| 점 | 초록 | A 이벤트 (Tic), 강한 신호 |
| 점 | 노랑 | A 이벤트 (Tic), 중간 신호 |
| 점 | 주황 | A 이벤트 (Tic), 약한 신호 |
| 점 | 진파랑 | C 이벤트 (Tac), 강한 신호 |
| 점 | 중간파랑 | C 이벤트 (Tac), 중간 신호 |
| 점 | 연파랑 | C 이벤트 (Tac), 약한 신호 |
| 수평선 | 초록 | A 이벤트 경계선 (Tic이 나타나야 할 예상 위치) |
| 수평선 | 파랑 | C 이벤트 경계선 (반주기 위치) |
| 배경 | 핑크 | 정상 작동 구간 |

**도메인 지식**:
기계식 시계의 이스케이프먼트는 Tic과 Tac 두 종류의 충격음을 냅니다. 이 소리의 위치와 강도가 일정하면 시계가 일관되게 작동하는 것입니다. Sound Print는 오실로스코프처럼 각 이벤트를 시각화하여 음향 패턴의 규칙성을 한눈에 파악합니다.

**해석 방법**:
- 점들이 두 수평선 근처에 빽빽하게 모여 있음 → 매우 일관된 Beat, 좋은 신호
- 점들이 세로로 퍼져 있음 → Beat 간 타이밍 불일치 (Jitter)
- 초록색 점이 없고 주황만 남음 → 마이크가 너무 멀거나 신호 약함
- 점들이 경계선에서 멀리 떨어짐 → 이스케이프먼트 조정 필요

## 3. Trace

![Trace](graphs/03-trace.png)

**한 줄 요약**: Rate와 진폭을 시간에 따라 선으로 그려, 시계가 시간이 지나면서 어떻게 변하는지 추적하는 그래프.

### 상단 패널 — Rate Error (오차율)

| 요소 | 색상 | 의미 |
|------|------|------|
| 선 | 파랑 | Rate 이동평균값 (s/d) |
| 텍스트 | 상단 헤더 | 현재 상태 요약 ("Rate & amplitude within normal limits" 등) |

- **X축**: 시간 (초). 윈도우 크기 조절 가능 (예: 30초)
- **Y축**: Rate Error (s/d). 0 = 완벽. 양수 = 빠름, 음수 = 느림

**도메인 지식**:
Rate는 Beat 주기를 측정하여 계산합니다. 28800 BPH에서 이론적 Beat 주기는 125ms. 실제 측정값이 125.001ms면 하루에 약 0.69초 느린 것. 메인스프링이 풀릴수록 토크가 줄어 Rate가 변합니다.

### 하단 패널 — Amplitude (진폭)

| 요소 | 색상 | 의미 |
|------|------|------|
| 선 | 초록 | 진폭값 (°) |
| 배경 | 초록 | 270–310°: Strong — 밸런스 휠이 충분히 스윙 |
| 배경 | 주황/노랑 | 220–270°: Acceptable — 허용 범위지만 주의 필요 |
| 배경 | 빨강 | < 220°: Needs Service — 즉각 점검 필요 |

**도메인 지식**:
진폭이 낮아지는 원인: ① 메인스프링 완전히 풀림 ② 윤활유 마름/경화 ③ 먼지·이물질 침투 ④ 파손된 부품. 새 태엽에서 오래된 태엽으로 갈수록 진폭이 서서히 줄어드는 것이 정상입니다.

**해석 방법**:
- 선이 수평으로 초록 구간 안에 → 정상
- 선이 완만하게 하강 → 태엽이 풀리는 중 (정상적 현상, 곧 다시 감아야 함)
- 선이 갑자기 뚝 떨어짐 → 기계적 문제 발생

## 4. Vario

![Vario](graphs/04-vario.png)

**한 줄 요약**: 세션 전체의 Rate와 진폭을 Min/Max/현재값/표준편차로 요약해 시계의 일관성을 직관적으로 보여주는 통계 뷰.

### 구성 요소

| 요소 | 의미 |
|------|------|
| 상단 대형 숫자 | 세션 경과 시간 (분:초) |
| **Rate** 섹션 헤더 | Min / Max / Now / σ 수치 (색상으로 구분) |
| 파란 화살표 | Min값과 Max값의 위치 (스케일 위에서의 범위 표시) |
| 빨간 화살표 | 현재(Now)값 위치 |
| 초록 배경 | 허용 범위 구간 |
| ✓ 체크마크 | 현재 값이 허용 범위 안에 있음 |
| **Amplitude** 섹션 | Rate와 동일한 구조, 단위만 ° |

- **X축 (Rate)**: s/d (초/일)
- **X축 (Amplitude)**: ° (도)
- **σ (표준편차)**: 측정값의 흩어짐 정도. 낮을수록 일관된 시계

**도메인 지식**:
Vario라는 이름은 "variation"에서 유래. 시계 측정의 표준 항목 중 하나로, 동일 포지션에서 반복 측정 시 결과의 분산을 나타냅니다. 고급 시계 제조사는 포지션별 Vario를 규격에 포함합니다.

**해석 방법**:
- Min과 Max 화살표 간격이 좁음 → 일관성 높음
- σ가 0에 가까움 → 매우 안정적
- 배경이 빨강으로 바뀜 → 허용 범위 초과, 조정 필요

## 5. Sequence (Multi-Position)

![Sequence](graphs/05-sequence.png)

**한 줄 요약**: 여러 포지션에서 측정한 결과를 하나의 표로 모아 포지션별 성능 차이를 비교하는 화면. 오른쪽에 레이더 차트가 함께 표시됩니다.

### 표 구조

**컬럼**:
- **Rate (s/d)**: 해당 포지션에서의 오차율
- **Beat (ms)**: 해당 포지션에서의 Beat Error
- **Ampl (°)**: 해당 포지션에서의 진폭

**행**:
- CH, CB, 9H, 6H, 3H, 12H, CU(R), CU(L), CD(R), CD(L): 각 측정 포지션
- 현재 활성 포지션은 보라색 강조 표시
- 미측정 포지션은 `—` 표시

**요약 행**:

| 행 | 의미 |
|----|------|
| **X (mean)** | 측정된 포지션 전체 평균 |
| **D (max−min)** | 포지션 간 최대 편차. 클수록 포지션에 따른 성능 차이가 큼 |
| **DVH** | 수직 포지션(9H/6H/3H/12H) 평균과 수평 포지션(CH/CB) 평균의 차이. 포이징(무게 균형) 이상 지표 |

**도메인 지식**:
중력 방향이 달라지면 밸런스 휠에 가해지는 힘이 달라집니다. 완벽하게 균형 잡힌 밸런스 휠은 포지션에 상관없이 동일한 Rate와 Amplitude를 보여야 하지만, 현실에서는 항상 차이가 납니다. DVH는 밸런스 휠의 무게 분포 불균형(포이징 오류)을 감지하는 핵심 지표입니다.

**경고 메시지** (하단):
- 허용 범위를 벗어난 포지션 수와 최악의 포지션을 알림
- 예: "6 of 6 positions out of tolerance — worst at CH (302°). Positional variance suggests poising / balance review."

## 6. Beat Scope

![Beat Scope](graphs/06-beat-scope.png)

**한 줄 요약**: 단일 Beat의 음향 파형을 약 20ms 창에서 확대하여, A이벤트와 C이벤트의 타이밍과 파형 형태를 직접 보는 화면.

### 메인 플롯

| 요소 | 색상/형태 | 의미 |
|------|-----------|------|
| 파형 | 노랑 | 고역통과 필터(HPF) 처리된 음향 신호의 절댓값 |
| 수직 점선 | 초록 | A 이벤트 감지 시각 (Tic) |
| 수직 점선 | 빨강 | C 이벤트 감지 시각 (Tac) |

- **X축**: 시간 (ms). 한 Beat 창 (예: 20ms)
- **Y축**: |Amplitude| — 신호 강도의 절댓값

**두 봉우리의 의미**:
- 첫 번째 봉우리 (초록선 근처) = Tic(A이벤트). 이스케이프먼트가 한 방향으로 풀리는 충격음
- 두 번째 봉우리 (빨강선 근처) = Tac(C이벤트). 반대 방향으로 풀리는 충격음
- 두 선 사이의 간격 = A→C 시간 간격 → 이 값으로 진폭과 Beat Error 계산

**하단 썸네일 스트립**:
최근 10 Beat의 파형을 작게 보여줍니다. 모든 썸네일이 같은 형태이면 일관된 시계. 썸네일 간에 형태가 다르면 Beat 간 불일치.

### 컨트롤
- **Single Beat / Average**: 단일 Beat 표시 또는 N개 겹쳐서 평균
- **Window (ms)**: X축 시간 범위
- **Latest beat**: 가장 최근 Beat 표시
- **Σ averaging**: 체크 시 여러 Beat를 겹쳐 노이즈 평균화

## 7. Beat Error

![Beat Error](graphs/07-beat-error.png)

**한 줄 요약**: Beat Error(Tic과 Tac의 간격 불균형)를 시간과 Beat 번호 기준으로 추적하는 이중 패널 디스플레이.

### 상단 패널 — Beat Error 시계열

| 요소 | 색상 | 의미 |
|------|------|------|
| 선 | 보라 | Beat Error 이동평균값 (ms) |
| 점 | 빨강 | Tic(A이벤트) 타이밍 오프셋 |
| 점 | 파랑 | Toc(C이벤트) 타이밍 오프셋 |
| 배경 | 초록 | 허용 범위 구간 (통상 0~0.5ms) |

- **X축**: 시간 (초). 슬라이딩 윈도우 (예: 30초)
- **Y축**: Beat Error (ms)
- **컨트롤**: Window 크기, Scale(Y축 범위)

**도메인 지식**:
Beat Error는 한 Beat 안에서 Tic→Tac과 Tac→다음Tic의 시간 비율 불균형입니다. 이상적으로는 A→C = C→A (각각 반주기). 불균형이 크면 시계가 "불규칙하게 틱틱" 소리가 나며, 포지션에 따라 Rate가 크게 달라집니다. 시계사는 이를 Beat Adjustment로 교정합니다.

### 하단 패널 — Timing Offset by Beat

| 요소 | 색상 | 의미 |
|------|------|------|
| 점 | 빨강 | Tic 타이밍 오프셋 (Beat 번호 기준) |
| 점 | 파랑 | Toc 타이밍 오프셋 (Beat 번호 기준) |

- **X축**: Beat 번호 (누적)
- **Y축**: 타이밍 오프셋 (ms)

**해석 방법**:
- 빨강과 파랑 점 모두 0에 붙어 있음 → Tic/Toc 완벽하게 균등
- 빨강은 0인데 파랑이 위로 치우쳐 있음 → Toc이 늦게 오고 있음 (Beat 조정 필요)
- 전체가 서서히 올라가거나 내려감 → Rate 드리프트

## 8. Long Term

![Long Term](graphs/08-long-term.png)

**한 줄 요약**: Rate, 진폭, Beat Error 세 가지를 하나의 시간축에 쌓아, 장시간 세션에서 시계의 변화를 한눈에 파악하는 화면.

**헤더**: 현재 Rate / Amplitude / Beat Error 수치 + 그래뉼래리티(측정 단위) 표시

### 상단 패널 — Rate (s/d)

| 요소 | 색상 | 의미 |
|------|------|------|
| 선 | 핑크/빨강 | 세션 전체 Rate 측정값 추이 |
| 수평 점선 | 어두운 빨강 | 세션 평균 Rate |

**도메인 지식**: 세션 초반에 Rate가 불안정한 것은 정상입니다. 새로 태엽을 감은 후 시계가 "워밍업"되면서 안정됩니다. 장시간 후 Rate가 다시 변하기 시작하면 태엽이 거의 풀린 것입니다.

### 중간 패널 — Amplitude (°)

| 요소 | 색상 | 의미 |
|------|------|------|
| 선 | 파랑 | 진폭 측정값 추이 |
| 음영 밴드 | 연파랑 | ±1σ 범위 — 측정값의 통상적인 변동 구간 |
| 수평 점선 | 파랑 | 세션 평균 진폭 |

**해석 방법**:
- 밴드가 좁고 수평 → 진폭이 일정, 건강한 시계
- 밴드가 넓음 → 매 측정마다 진폭이 크게 흔들림 (불안정)
- 갑자기 하락 → 기계적 문제 (윤활유 고착, 이물질 등)

### 하단 패널 — Beat Error (ms)

| 요소 | 색상 | 의미 |
|------|------|------|
| 선 | 초록 | Beat Error 측정값 추이 |
| 음영 밴드 | 연초록 | ±1σ 범위 |
| 수평 점선 | 초록 | 세션 평균 Beat Error |

세 패널이 동일한 시간축을 공유하므로, "이 시각에 진폭이 떨어지면서 Rate도 변했다"처럼 상관관계를 직접 확인할 수 있습니다.

## 9. Escapement

![Escapement](graphs/09-escapement.png)

**한 줄 요약**: 단일 Beat의 A→C 간격을 마이크로초 수준으로 측정하고, 이스케이프먼트 동작의 일관성을 정량화하는 정밀 분석 화면.

### 플롯 요소

| 요소 | 색상 | 의미 |
|------|------|------|
| 파형 | 회색 | 음향 신호 (정규화) |
| 수직 실선 | 초록 | A 이벤트 (T1, Tic) — 0ms 기준점 |
| 수직 실선 | 빨강 | C 이벤트 (T3, Tac) — Peak 또는 Onset 기준 |
| 음영 구간 | 주황/노랑 | A→C 측정 구간 |
| 레이블 | 검정 | Δ값 (A→C 간격, ms) |

- **X축**: A 이벤트로부터의 경과 시간 (ms). 음수 구간 = A 이벤트 이전
- **Y축**: 정규화된 진폭 (0~1)

### 헤더 정보

- **C reference**: Peak (파형 최고점) 또는 Onset (신호 시작점) — 선택 가능
- **A→C interval**: 현재 측정된 A→C 간격 (ms)
- **σ peak / σ onset**: 최근 N Beat에서 각 기준점의 표준편차 (ms)
- **"Onset more stable"**: σ onset < σ peak일 때 표시 — 어느 기준이 더 일관적인지 안내

**도메인 지식**:
이스케이프먼트 분석은 시계 수리사가 진단할 때 핵심 항목입니다. A→C 간격이 클수록 진폭이 크고(좋음), 일관성이 높을수록(σ 낮음) 이스케이프먼트가 정확합니다. σ가 0.02ms 이하면 매우 우수한 수준입니다.

**파형 형태 해석**:
- A 이벤트 이후 조용한 구간(노이즈 없음) + C에서 날카로운 스파이크 → 이스케이프먼트 정상 작동
- 조용한 구간에 추가 충격(범프) 발생 → 이스케이프먼트 결함 (partial unlocking 등)
- C 이벤트 스파이크가 뭉툭함 → 부품 마모

## 10. Spectrogram

![Spectrogram](graphs/10-spectrogram.png)

**한 줄 요약**: 음향 신호를 주파수×시간 2D 이미지로 변환하여, 시계에서 어떤 주파수 성분이 언제 발생하는지 시각화하는 화면.

### 플롯 요소

| 요소 | 의미 |
|------|------|
| **X축** | 시간 (ms). 한 Beat 창 (예: 200ms) |
| **Y축** | 주파수 (Hz). 0~20,000 Hz |
| **색상** | 노랑 = 크다 (약 −40 dBFS), 시안 = 중간, 보라 = 작다 (약 −65 dBFS) |
| **True Peak 미터** | 상단 수평 바. 전체 신호 레벨 (dBFS). 초록 = 적정 |

**무엇이 보이는가**:
- **두 개의 밝은 수직 띠** = A 이벤트(Tic)와 C 이벤트(Tac). 기계적 충격은 20kHz까지 광대역 에너지를 방출
- **띠가 선명하고 좁음** = 충격이 순간적이고 날카로움 (건강한 이스케이프먼트)
- **배경이 보라** = 이벤트 사이에 잡음 없음

**도메인 지식**:
Reassigned Spectrogram은 일반 STFT보다 시간-주파수 해상도가 높습니다. 기계식 시계처럼 짧은 충격음이 반복되는 신호를 분석하는 데 특히 적합합니다.

**이상 징후 감지**:
- A와 C 사이에 추가 수직 띠 → 이스케이프먼트 추가 충격 (부품 결함)
- 특정 주파수에서 수평 띠 → 공진하는 부품 (풀린 나사, 마모된 기어 등)
- 미터가 클리핑(빨강) → 마이크 너무 가깝거나 게인 너무 높음

## 11. Waveform

![Waveform](graphs/11-waveform.png)

**한 줄 요약**: 연속된 Beat를 세로로 쌓아 나란히 비교하여, Beat 간 파형의 일관성과 A→C 타이밍 변이를 직접 보는 화면.

### 각 패널 구조

| 요소 | 색상 | 의미 |
|------|------|------|
| 파형 | 노랑 | HPF 처리된 음향 신호 (고역통과 필터 적용) |
| 수직선 | 파랑 | A 이벤트 위치 (기준점, 모든 패널에서 정렬됨) |
| 스파이크 | 노랑 | C 이벤트 위치 |
| 패널 제목 | — | Beat 번호 + 해당 Beat의 t_AC 값 (ms) |

- **X축 (하단)**: A 이벤트 이후 경과 시간 (ms)
- **X축 (상단)**: A 이벤트 이후 밸런스 휠 각도 (°). Lift Angle까지의 구간 표시
- **Y축**: HPF 진폭

**도메인 지식**:
t_AC(A→C 시간)는 밸런스 휠이 A에서 C까지 스윙하는 데 걸리는 시간입니다. 이 값이 일정하면 이스케이프먼트가 일관되게 작동하는 것. 상단 각도 축의 Lift Angle(예: 52°)은 이스케이프먼트가 팔레트 포크를 해제하는 시점의 밸런스 휠 각도로, 시계 모델마다 다릅니다.

**헤더 요약**:
- 전체 세션의 t_AC 최솟값 / 최댓값 / σ
- σ가 작을수록 Beat 간 일관성이 높음

**해석 방법**:
- 모든 패널에서 노랑 스파이크가 같은 위치 → 매우 일관된 이스케이프먼트
- 스파이크 위치가 패널마다 조금씩 다름 → Beat 간 타이밍 변이 (t_AC 산포)
- 한 패널에서 스파이크가 없음 → 해당 Beat C이벤트 감지 실패

## 12. Sweep

![Sweep](graphs/12-sweep.png)

**한 줄 요약**: 여러 Beat를 하나의 긴 시간축 위에 펼쳐 보여주는 오실로스코프 스타일 화면. 시계의 Rate를 직관적으로 확인할 수 있습니다.

### 플롯 요소

| 요소 | 색상 | 의미 |
|------|------|------|
| 스파이크 쌍 | 핑크/마젠타 | 하나의 Beat. 첫 스파이크 = A(Tic), 두 번째 = C(Tac) |
| 배경 | 흰색 | 신호 없는 구간 |

- **X축**: Sweep 시간 (ms). 설정한 tick 수에 따라 범위 결정 (예: 4 ticks = ~500ms)
- **Y축**: |Amplitude| — 신호 강도
- **헤더**: Daily Rate / Amplitude / Beat Error / BPH 요약
- **X축 레이블**: "stable pattern = on-rate, drifting = fast/slow"

**Sweep length 설정**:
몇 개의 Beat(tick)를 한 화면에 보여줄지 설정. 4 ticks = 4 Beat 쌍 = 약 500ms (28800 BPH 기준).

**도메인 지식**:
전통적인 기계식 시계계기(타이머기)에서 사용하던 표시 방식입니다. 종이 테이프에 점을 찍어 드리프트를 보는 방식과 동일한 개념. 스파이크 쌍이 화면 왼쪽으로 몰리면 빠른 것, 오른쪽으로 밀리면 느린 것.

**해석 방법**:
- 스파이크 쌍들이 등간격으로 안정적 → 시계가 on-rate
- 연속된 스파이크 쌍이 점점 왼쪽으로 쏠림 → 시계가 빠르게 가는 중
- 쌍 내부 두 스파이크 간격이 불규칙 → Beat Error 존재

## 13. Filter Scope

![Filter Scope](graphs/13-filter-scope.png)

**한 줄 요약**: 하나의 Beat 사이클(125ms)을 Raw 신호부터 최종 감지 신호까지 4단계 필터 파이프라인으로 쌓아 보여주는 진단 화면.

**헤더**: Beat 사이클 길이(ms) / 샘플 수 / 샘플레이트 / BPH

### 4개 패널

| 패널 | 필터 단계 | 설명 |
|------|-----------|------|
| **Raw** | HPF 적용 | 고역통과 필터만 적용한 원신호. 이벤트가 날카로운 스파이크로 나타나지만 노이즈가 섞여 있음 |
| **Smoothed** | HPF + 이동평균 | 이동평균으로 평활화. 이벤트 형태가 더 명확해지고 노이즈 감소. 양(+)과 음(−) 모두 표시 |
| **Envelope** | HPF + 포락선 | 스무딩 신호의 절댓값(포락선). 음수 부분을 제거하고 에너지 윤곽만 표시 |
| **Upper Envelope** | HPF + 상위 포락선 | 단 하나의 선명한 양의 스파이크만 남음. 실제 Beat Detector가 사용하는 신호 |

- **X축**: 시간 (ms). 모든 패널 동일한 범위 (0~125ms, 한 Beat 사이클)
- **Y축**: 각 단계별 진폭 (정규화)

**도메인 지식**:
시계 음향 신호는 충격음 외에도 공진, 진동, 외부 잡음이 섞입니다. 단순 임계값 감지로는 오감지(False Positive)가 발생합니다. 4단계 필터 파이프라인은 불필요한 성분을 제거하고 Beat 이벤트만 추출합니다. High Pass Cutoff 파라미터(Misc. Parameters)가 Raw 단계에서의 필터 차단 주파수를 결정합니다.

**개발 활용**:
이 화면으로 필터 파라미터를 튜닝했습니다. Upper Envelope에서 Beat 이외의 스파이크가 나타나면 파라미터 조정이 필요합니다.

## 14. Radar Chart (보너스)

![Radar Chart](graphs/14-radar-chart.png)

**한 줄 요약**: 여러 포지션에서 측정한 진폭(또는 Rate)을 방사형 차트로 시각화하여, 포지션 간 불균형을 즉시 파악하는 보너스 화면.

### 플롯 요소

| 요소 | 색상 | 의미 |
|------|------|------|
| 점 | 빨강 | 해당 포지션의 측정값 |
| 폴리곤 | 파랑 (채움) | 측정값을 연결한 다각형 |
| 원형 점선 | 초록 | 허용 범위 경계 (안쪽 = 최소, 바깥쪽 = 목표) |
| 레이블 | 검정 | 포지션 이름 (CH, CB, 9H 등) |
| 눈금 | 회색 | 동심원 = 수치 스케일 (예: 200, 225, 250, 275, 300) |

- **메트릭 선택**: 상단 드롭다운으로 Amplitude(°) 또는 Rate(s/d) 전환
- **Sequence 탭 내 미니 레이더**: 테이블과 나란히 표시 (더 작음)
- **Radar 탭**: 전체 화면으로 더 크게 표시

**도메인 지식**:
이상적으로 완벽하게 포이징된 밸런스 휠은 모든 포지션에서 동일한 진폭을 보여 레이더 차트가 완전한 원이 됩니다. 현실에서는 포이징 오류, 밸런스 스프링의 비대칭성, 마찰 등으로 인해 항상 비대칭이 발생합니다. 수평 포지션(CH/CB)과 수직 포지션(9H~12H)의 차이는 중력에 의한 구조적 효과이며, DVH로 정량화됩니다.

**해석 방법**:
- 폴리곤이 초록 외원에 균일하게 붙어 있음 → 포지션 간 편차 작음, 잘 조정된 시계
- 특정 방향으로 폴리곤이 안쪽으로 당겨짐 → 해당 포지션에서 진폭 낮음 (포이징 문제 의심)
- 수직 포지션들(9H/6H/3H/12H) 전체가 수평(CH/CB)보다 낮음 → DVH 큼 → 밸런스 휠 질량 분포 불균형
- 하단 경고 메시지: 허용 범위 초과 포지션 수, 최악 포지션, 추정 원인
